### Ответы на контрольные вопросы

**1. Какие типы меню существуют в Android? Опишите их назначение.**

- **OptionsMenu (главное меню)** — вызывается нажатием на кнопку меню в ActionBar (три точки). Используется для глобальных действий приложения (настройки, поиск, выход).
- **ContextMenu (контекстное меню)** — появляется при долгом нажатии (long click) на элементе интерфейса. Предоставляет действия, специфичные для выбранного элемента.
- **PopupMenu (всплывающее меню)** — привязано к определённому View и появляется по нажатию (обычно на кнопку).

**2. Как создать главное меню (OptionsMenu)? Какие методы необходимо переопределить в Activity?**

Для создания главного меню необходимо:
1. Создать XML-файл в папке `res/menu/` с описанием пунктов меню
2. Переопределить метод `onCreateOptionsMenu()` для загрузки меню:
```kotlin
override fun onCreateOptionsMenu(menu: Menu): Boolean {
    menuInflater.inflate(R.menu.main_menu, menu)
    return true
}
```
3. Переопределить метод `onOptionsItemSelected()` для обработки выбора пунктов

**3. Для чего используется атрибут app:showAsAction? Какие значения он может принимать?**

Атрибут `app:showAsAction` определяет, будет ли пункт отображаться непосредственно в ActionBar или будет скрыт в выпадающем меню:
- `ifRoom` — отображать, если есть место
- `never` — всегда скрывать в меню
- `always` — всегда показывать (не рекомендуется, так как может не хватить места)
- `withText` — показывать вместе с текстом (если есть иконка)

**4. Как зарегистрировать View для контекстного меню? В каком методе это обычно делается?**

Регистрация выполняется с помощью метода `registerForContextMenu(View view)`. Обычно это делается в методе `onCreate()`:
```kotlin
val button = findViewById<Button>(R.id.btnAddFigure)
registerForContextMenu(button)
```

**5. В чём разница между методами onCreateContextMenu и onContextItemSelected?**

- **`onCreateContextMenu()`** — вызывается при открытии контекстного меню, используется для создания и настройки меню (добавление пунктов, установка заголовка)
- **`onContextItemSelected()`** — вызывается при выборе пункта меню пользователем, используется для обработки выбранного действия

**6. Как создать контекстное меню динамически (программно), без использования XML-ресурса?**

В методе `onCreateContextMenu()` можно добавлять пункты программно с помощью метода `menu.add()`:
```kotlin
override fun onCreateContextMenu(menu: ContextMenu, v: View, menuInfo: ContextMenuInfo) {
    super.onCreateContextMenu(menu, v, menuInfo)
    menu.setHeaderTitle("Выберите фигуру")
    menu.add(0, 1, 0, "Квадрат")
    menu.add(0, 2, 1, "Круг")
}
```

**7. Что возвращают методы onOptionsItemSelected и onContextItemSelected? Что означает возврат true?**

Методы возвращают `boolean` значение:
- `true` — означает, что обработка выполнена и событие дальше не передаётся
- `false` — событие передаётся для дальнейшей обработки (например, в суперкласс)

**8. Как определить, для какого именно элемента было вызвано контекстное меню, если зарегистрировано несколько View?**

Можно использовать параметр `v` в методе `onCreateContextMenu()` или получить информацию через `menuInfo`. Для стандартных View можно сравнивать идентификаторы:
```kotlin
override fun onCreateContextMenu(menu: ContextMenu, v: View, menuInfo: ContextMenuInfo) {
    when (v.id) {
        R.id.button1 -> menu.setHeaderTitle("Меню для кнопки 1")
        R.id.button2 -> menu.setHeaderTitle("Меню для кнопки 2")
    }
}
```
